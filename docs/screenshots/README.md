# Screenshots

These are intentionally left for you to capture on your own machine — the build
environment this project was assembled in has no reachable headless browser, so
generating real screenshots here would have meant faking them, which this
project's whole ethos rules out.

To capture them (backend running at http://127.0.0.1:8000):

1. `landing.png` — the landing page hero
2. `dashboard.png` — main dashboard with map + KPIs
3. `ward-detail.png` — a ward's explainability panel
4. `simulator.png` — What-If simulator with a result
5. `data-sources.png` — provider status + upload

The demo script in `docs/DEMO_SCRIPT_V2.md` walks through each of these views.
